编程与刷牙

如果把口腔卫生比做一个程序，用牙习惯就是代码质量，蛀牙就是BUG，良好的代码质量可以减少BUG产生的几率，本文通过类比的方式，向大家介绍保护牙齿的心得，如果你可以写出优美的程序，那也一定能拥有健康的牙齿。

开发工具：牙刷

选择一款合适的牙刷是提升口腔质量的第一步，就像写代码需要一款趁手的编辑器一样。考虑到清洁最深处的犄角旮旯，牙刷的灵活性是很重要的，刷头应该尽量选择小一点的，刷柄也不建议特别粗。刷毛应选择比较细的，可以清理到龈沟（牙齿根部与牙龈结合的那条小沟沟），软硬度的话根据个人喜好，硬度越高清洁效果越好，相对的对牙齿的损伤就越大。也可以选择现代化的电动牙刷，常见的电动牙刷分为旋转式和声波式，相比传统手动牙刷来说电动牙刷的效率更高，同样时间可以达到数十倍的清洁效果，相对的使用不当对牙齿牙龈的损伤也更大，比较好的产品会有压力提示，当然价格也更高，一支较好的牙刷可能几百元到上千元。这里有个误区是，很多人觉得使用电动牙刷就能解决所有问题了，电动牙刷只是解决了效率问题，就像用记事本写出的代码质量未必比SublimeText写出的差，只是后者的体验更好而已。

基础模块：含氟牙膏

基础模块的质量决定了业务的稳定性，也是一个项目成败的关键，所以牙膏的选择是非常重要的，牙膏最基本的功能是通过细小的颗粒与牙齿表面摩擦，从而得到去除牙菌斑的目的，然后根据不同的需要还加入了美白、消炎、清口气等功效。针对蛀牙来说，含氟牙膏可以有效起到预防的作用，蛀牙形成的原理通俗来讲就是细菌一点一点把牙齿表面侵蚀了，开始是一个个蛀蚀点，久而久之蛀蚀点越来越大，最后变成蛀洞，含氟牙膏被证实可以修复牙釉质。关于含氟牙膏致癌的言论是没有科学依据的，少量的氟化物对人体是无害的，长期过量摄入氟化物才会产生毒素。

设计模式：Bass刷牙法

刷牙的相当于项目中真正我们写的一行行代码，代码是要讲究结构和设计的，所以刷牙也不是一通乱刷的，只有遵循正确的刷牙方式，才能最大程度的发挥出牙刷和牙膏的功效，得到清洁牙齿的目的。Bass刷牙法是目前美国牙科协会推荐的一种刷牙方式，主要针对龈沟这个比较容易积攒细菌的地方清理

巴斯刷牙法：又称龈沟清扫法或水平颤动法。 选择软毛牙刷，将牙刷与牙长轴呈45°角指向根尖方向（上颌牙向上，下颌牙向下），按压龈－牙交界区，使刷毛一部分进入龈沟，一部分铺于龈缘上，并尽可能伸入邻间隙内，用轻柔的压力，使刷毛在原位作前后方向短距离的水平颤动10次。颤动时牙刷移动仅约1mm，每次刷2～3个牙。在将牙刷移到下一组牙时，注意重叠放置
科学建议每天至少2次刷牙，尤其临睡前一次是非常重要的。关于Bass刷牙法的具体操作过程，可以搜索视频较为直观的学习，平时刷牙时要注意多加练习。

编译：漱口水

要让代码run起来就不能有语法错误，运行之前必须找到并且修复这些语法错误，就像刷牙过程中，难免会疏忽某些地方，使用漱口水就可以降低口腔内整体的细菌数。关于漱口水到底是否有用，在知乎上已经讨论过，绝大部分人认为漱口水是可以起到杀菌和清新口气的作用的，由于漱口水主要成分是酒精，对于酒精比较敏感的人的体质来说，漱口水可能会带来一些副作用，目前也有一些不含酒精成分的漱口水可以选择，总体来说漱口水有一定效果，而且没有什么危害，建议使用。

希望大家通过对本文的阅读后重视对口腔的护理，不管是刷牙还是写代码，都是一门科学，对待科学就要讲究方式，祝大家写得一手好代码的同时也有一口好牙齿。

SourceTree 压缩多次commit

在git的使用过程中，有时会进行一些保存性的commit，这样会生成一些无意义的记录，如：

为了避免这种情况的发生，可以使用git为我们提供的交互式变基功能，使用命令行操作的方法可以参考：http://blog.csdn.net/itfootball/article/details/44154121

使用SourceTree进行操作：

Step 1: 右键要合并到的那条commit -> 交互式变基

Step 2: 选中最后一条被合并的commit -> 点击”用以前的提交来squash”

Step 3: 然后将要合并的commit一条条拖进分组

Step 4: 右击最后一条commit点击”编辑信息” -> 修改commit信息 -> 确认

完成:

这种方法固然方便，但是要注意，这种操作必须是在没有push之前完成，如果已经push过一些commit，再进行压缩的话是无法再push的，此时只能通过force push强推

所以比较保险的办法是——新建分支 只要确保这条分支是属于你个人的，那么在分支内进行任何操作都是安全的，再完成功能开发后，进行一次压缩，再合并回主分支，既安全生成的历史又美观

原生js实现$.each()方法

假设页面上有这些P元素&lt;p&gt;1&lt;/p&gt; &lt;p&gt;2&lt;/p&gt; &lt;p&gt;3&lt;/p&gt; &lt;p&gt;4&lt;/p&gt;

$('p').each(function(){
  console.log(this.innerHTML);
});
`</pre>

## 如何用原生的js实现上面这个each方法呢？

首先要理解这段代码的原理，这段代码可以分成3个部分，`$('p')`、`each()`和一个callback函数, 然后再扩展object对象，在扩展方法中调用`each()`。

### 首先来看`$('p')`

`$()`其实是一个函数
<pre>`function $(){
  return;
}
`</pre>
想要用这个函数返回页面所有的p元素可以这样写
<pre>`function $(tag){
  var result;
  if(typeof tag ==='string')
    result = document.getElementsByTagName('p');

    return result;
}

console.log($('p').length); // 4
`</pre>
OK，搞定！jQuery的选择器的原理也不是很难嘛～

### 再来看each()

`each()`要做的事情就是传入一个数组对象，然后枚举出它每个元素，不就是循环嘛！So easy
<pre>`function each(arr){
  if(typeof arr === 'object'){
    for (var i = 0; i &lt; arr.length; i++){
    }
  }
}
`</pre>

#### 万事具备只差callback

callback只是一个函数，再每次遍历的时候去执行这个函数，所以只许对each函数略加修改即可
<pre>`function each(arr, callback){
  if(typeof arr === 'object'){
    for (var i = 0; i &lt; arr.length; i++){
      callback();
    }
  }
}
`</pre>
OK！劳资写好了，根本没有难度！跑下看看！
<pre>`var $p = $('p');

each($p, function(){
  console.log(this.innerHTML);
});
`</pre>
此时打开控制台看到的结果：
![](http://ww2.sinaimg.cn/large/80a06c8djw1esizdsk1h6j210q08s427.jpg)

![](http://ww4.sinaimg.cn/large/80a06c8djw1esizk2j6w6j20g40g4gm8.jpg) **wtf？？**
为什么`this`是这玩意？！
好吧，这里的this属于全局性调用，也就是指向了window，[关于this的用法请看这里](http://www.ruanyifeng.com/blog/2010/04/using_this_keyword_in_javascript.html "undefined")，**那么我要如何重新指定this呢？**

### 改变this指向的神器：`Call()`

> 当调用一个函数时，可以赋值一个不同的 this 对象。this 引用当前对象，即 call 方法的第一个参数。

通过 call 方法，你可以在一个对象上借用另一个对象上的方法，比如Object.prototype.toString.call([])，就是一个Array对象借用了Object对象上的方法。
简单来说，call方法就是在调用函数的同时，指定该函数的this指向call的第一个参数，并且传递若干个参数。
OK!有办法了，还等什么！来改造一下each函数
<pre>`function each(arr, callback){
  if(typeof arr === 'object'){
    for (var i = 0; i &lt; arr.length; i++){
      callback.call(arr[i], i, arr[i]);
      }
    }
}

var $p = $('p');

each($p, function(){
  console.log(this.innerHTML);
});
`</pre>
![](http://ww3.sinaimg.cn/large/80a06c8djw1esj37438qlj2122094tah.jpg)
COOL！这就是俺要得效果～

### 最后一步

我想要把each方法直接作为对象的方法，这里要使用prototype对object对象进行扩展，[对prototype的使用请点击此处](http://blog.93html.com/javascripe-object-function-and-class-function/ "undefined")
<pre>`Object.prototype.each = function(func) {
  return each(this, func);
};
`</pre>
Allright! 我也可以写出高逼格的代码啦～
![](http://ww4.sinaimg.cn/large/80a06c8djw1esj3gtimbqj20jg0jgjsi.jpg)
<pre>`$('p').each(function() {
  console.log(this.innerHTML);
});
