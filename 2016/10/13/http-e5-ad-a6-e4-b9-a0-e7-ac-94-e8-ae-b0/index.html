2. URI与资源

URL 是因特网资源标准化的名称

URI 是一个通用概念，由2个主要子集URL 和URN 组成



URL 描述资源位置 URN 描述资源名称

URL 语法

<scheme>://<user>:<password>@<host>:<port>/<path>;<params>?<query>#<frag>




scheme 协议 大小写不敏感 host&port 主机地址
user&password ftp path 资源位置
params 额外信息 query 缩小请求范围
frag 指向文档片段


相对URL 相对一个基础(base) URL

URL 编码机制 % + ASCLL


常见方案

HTTP
HTTPS MAILTO
FTP RTSP 流媒体
RTSPU 使用UDP FILE
NEWS 新闻 需指定Nntp服务器 TELNET 交互式业务

URN 一种永久资源定位方式


3. 报文




流入服务器，从服务器流出


发送者称为上游，接受者称为下游，所有报文都向下游流动


报文由起始行(start line) 首部(header) 主体(body) 三部分构成


请求文格式:

<method> <request-url> <version>

<headers>


<entity-body>


响应文格式(仅起始行不同):

<version> <status> <reason-phrase>

<headers>


<entity-body>
